操作指南

操作指南

由于生存手册项目于 2024 年 3 月重构，站点生成系统采用新架构，故发布流程等亦已改变。此指南旨在帮助站点维护者快速入手。

⚠️

注意： 请不要修改本指南的内容，若认为有问题，请联系管理人员。

技术架构

生存手册为纯静态网页项目，采用 Next.js (opens in a new tab) 进行站点生成，MDX (opens in a new tab) 作为书写语言，使用 Render (opens in a new tab) 进行服务，采用 Cloudflare (opens in a new tab) 作为 CDN。

为了确保安全性和减少复杂性，生存手册的内容管理已从 Bitbucket 转向私有服务。本站点的内容依旧遵循 GNU FDL (opens in a new tab) 协议，你依旧可以获取全部的源码。

如果你希望向站点贡献内容，或提供建议，请通过发送邮件的方式，也可以选择在相关平台留言。详情请查看主页的内容。

安装

安装开发环境

若想要顺利地运行生存手册的开发环境，你需要以下软件包：

  • 较新版本的 Bash (opens in a new tab) 或 Zsh (opens in a new tab)：用于基础环境的操作，一般无须额外安装
  • Node.js (opens in a new tab) v20 及以上：是一个 JavaScript 运行时，用于运行整个项目
  • 较新版本的 yarn (opens in a new tab)：Node.js 的包管理系统

以下介绍安装方式，在安装前，请确保能够链接国际互联网。

macOS

安装 Node.js

使用 brew 安装之：

brew install node@21

安装完毕后，请确认命令 node 存在且能够执行。运行 node --version，若有以下信息则安装成功（版本可能有所不同）:

-> % node --version
v21.10.0

安装 yarn

使用 brew 安装 yarn：

brew install yarn

安装完毕后，使用 yarn -v 确认（版本可能有所不同）：

-> % yarn -v
1.22.21

安装和设置 git

你需要安装和设置 git，使用 brew 安装完成以后，使用以下命令设置用户名和伊妹儿，它们将公开显示：

git config --global user.name "Your Name"
git config --global user.email you@example.com

Linux 用户，请自行使用对应发行版的包管理工具安装 Node.js，余步相同，略。

在搭建完毕后，请获取源码分发包并解压，得到的目录结构如下：

    • next.config.js
    • package.json
    • theme.config.jsx
    • yarn.lock

    • 其中：

    • .next是 nextjs 的生成文件
    • node_modules是 Node.js 的模块和依赖文件
    • out是最终构建的手册内容
    • pages是手册的源文件
    • public是其他资源，例如图片、视频、文件等

    接下来初始化项目，此操作仅需要执行一次。在上示目录中打开终端，运行 yarn install，根据网速的情况，需要等待一段时间，若最终显示以下信息则安装完毕：

    创建新文章

    /pages 是整个手册的根目录，其目录结构也就是左侧站点导航栏的结构。在每一个子目录中，都有一个 _meta.json 文件，其控制了磁盘上的文件名和实际显示的对应关系，还提供了其他细节控制。以 /pages/about 目录为例：

      • _meta.json
      • disclaimer.mdx
      • help.mdx
      • license.mdx

    • 这个子目录一共包含三个文章，都以 .mdx 为扩展名。以 .mdx 为扩展名的文件是扩展的 Markdown 格式，其格式与 Markdown 基本一致，但有一些添加的功能。打开 _meta.json，其内容如下：

    {
    "license": "许可证",
    "disclaimer": "免责声明",
    "help": "操作指南"
}

    由于框架的限制，磁盘上文章文件名不能是中文，因此必须有一个方法记录磁盘上英文命名的文件对应目录项目是什么，这就是 _meta.json 的主要作用。关于 json 文件的格式，无须太多了解就能够使用，记得引号和逗号，建议使用具有 自动格式化功能的工具编辑。文件中，左侧的是本目录下不带扩展名的文件名，右侧是网页上实际显示的名字。对于每一个文章，都需要在它所属的目录的那个 _meta.json 文件中建立这条记录。 否则就会显示文件名（英文名字）。对于目录，则是在它的上一层的 _meta.json 中键入信息。例如 /pages/life 在磁盘上的目录名是 life，但网页上显示的是生活指南，这个对应关系就放在其上一层中，也就是 /pages/_meta.json:

    ...
    "life": "生活指南",
...

    在实践中，建议仅使用大小写英文字母、数字、下划线和连字符命名文件，避免使用特殊符号和空格。

    假如你需要新建一个文章，首先你需要确认你是要在哪一个目录建立，之后在那里创建这个文件 your-file.mdx，接着在新文件的这个目录的 _meta.json 中新增对应关系记录，例如：

    {
    "license": "许可证",
    "disclaimer": "免责声明",
    "help": "操作指南",
    "your-file": "我的大作"
}

    接下来就可以编辑和预览了。

    假如你是要新建一个目录，那么你需要在这个目录的父目录中的 _meta.json 中键入信息，你还需要在新建的目录中创建 _meta.json。

    编辑文章

    注意事项

    对于要添加的图片等内容，使用标准的 Markdown 写法，你的图片应该放在 /public 目录中，引用时，去掉 /public 即可，例如，你在有一个 /public/hello.webp 的图片，你应该这样使用之：

    ![我是图片](/hello.webp)

    [如何上网](/guide/net)

    删除已有的文章

    注意事项与新建文章相同，你需要将对应的 _meta.json 也进行修改，避免不一致。

    另外，注意备份和版本管理，避免意外发生。
